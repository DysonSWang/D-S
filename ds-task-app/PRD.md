## 1. 产品概述

### 1.1 产品定位

一款面向**亲密关系成长**的任务打卡管理 Web 应用，帮助用户与伙伴建立更有序、更有仪式感的互动关系，促进自我提升和习惯养成。

> **定位说明**: 
> - 对外定位：「亲密关系成长工具」或「习惯养成打卡」
> - 核心场景：伴侣/伙伴之间的任务互动、习惯监督、成长记录
> - 价值主张：让关系更有仪式感，让成长更有动力

### 1.2 产品目标

- ✅ 提供仪式感的**关系缔结流程**，建立正式的伙伴关系
- ✅ 完善的任务体系，支持多种任务类型和发布方式
- ✅ 虚拟奖励系统，增强互动趣味性
- ✅ 个人档案与偏好建设，帮助双方更好了解彼此
- ✅ 安全边界保护，确保关系健康可持续
- ✅ **用户高度自主**，可随时调整或退出关系

### 1.3 目标用户

| 用户类型 | 描述 |
|---------|------|
| 引导者 (Guide) | 希望帮助伙伴成长，布置任务给予鼓励的一方 |
| 成长者 (Grower) | 希望自我提升，接受任务监督并获得奖励的一方 |
| 平台管理员 | 负责平台运营、内容审核、用户管理 |

> **用词说明**: 
> - 使用「引导者/成长者」替代敏感词汇
> - 强调「成长」「提升」「互助」的正向价值
> - 关系本质是**平等自愿**的伙伴关系

### 1.4 核心场景

```
场景 1：关系缔结仪式
引导者发送邀请码 → 成长者输入邀请码 → 双方确认约定 → 关系建立

场景 2：日常任务
引导者布置任务 → 成长者接收通知 → 完成任务打卡 → 引导者审核 → 发放奖励

场景 3：自主挑战
成长者抽取随机任务 → 完成任务 → 获得额外奖励

场景 4：成就展示
成长者查看自己的成就墙 → 分享成就 → 获得伙伴鼓励
```

### 1.5 产品原则

| 原则 | 说明 |
|------|------|
| **用户自主** | 用户可随时调整关系、退出关系，系统不强制绑定 |
| **平等自愿** | 所有互动基于双方自愿，系统提供协商工具 |
| **安全优先** | 内置安全词、边界设置、紧急暂停等保护机制 |
| **正向引导** | 强调成长、提升、习惯养成，避免负面内容 |
| **合规经营** | 严格内容审核，禁止违法违规内容 |

### 1.6 技术约束

| 项目 | 选择 |
|------|------|
| 部署方式 | 云服务器（Linux） |
| 数据库 | SQLite（初期） |
| 图片存储 | OSS 对象存储 |
| 通知方式 | 站内信 |
| 开源协议 | MIT / Apache 2.0 |
| 预算 | 开源免费（仅云服务器 + OSS 费用） |

## 2. 用户角色

### 2.1 角色权限矩阵

| 功能模块 | 管理员 | 引导者 | 成长者 |
|---------|-------|--------|--------|
| 用户管理 | ✅ | ❌ | ❌ |
| 内容审核 | ✅ | ❌ | ❌ |
| 系统配置 | ✅ | ❌ | ❌ |
| 关系缔结/解除 | ❌ | ✅ | ✅ |
| 布置任务 | ❌ | ✅ | ❌ |
| **自主领取任务** | ❌ | ❌ | ✅ |
| 任务打卡 | ❌ | ❌ | ✅ |
| 审核打卡 | ❌ | ✅ | ❌ |
| 发放奖励 | ❌ | ✅ | ❌ |
| 查看统计 | ✅ | ✅（自己的伙伴） | ✅（自己的） |
| 私信 | ✅ | ✅ | ✅ |

> **关键变化**: 
> - 成长者可**自主领取任务**，不完全依赖引导者布置
> - 双方均可**主动解除关系**，无需对方同意
> - 强调「伙伴」而非「主从」关系

### 2.2 关系模式

系统支持**两种关系模式**，用户可自由选择：

```
┌─────────────────────────────────────────────────────────────┐
│  模式 A: 成长伙伴模式（默认推荐）                            │
├─────────────────────────────────────────────────────────────┤
│  - 关系定位：平等的成长伙伴                                  │
│  - 任务来源：双方协商 + 自主领取                            │
│  - 审核机制：互相鼓励，非强制审核                            │
│  - 适合场景：情侣、朋友、学习搭档                            │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  模式 B: 引导成长模式                                       │
├─────────────────────────────────────────────────────────────┤
│  - 关系定位：一方引导，一方成长                              │
│  - 任务来源：引导者布置 + 成长者自选                        │
│  - 审核机制：引导者审核打卡，发放奖励                        │
│  - 适合场景：希望有明确角色分工的伙伴                        │
└─────────────────────────────────────────────────────────────┘
```

> **设计说明**: 
> - 默认推荐「成长伙伴模式」，更平等、更合规
> - 「引导成长模式」保留原有核心功能，但用词中性化
> - 用户可在关系设置中切换模式

### 2.3 关系生命周期

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  邀请    │ →  │  缔结    │ →  │  互动    │ →  │  归档    │
│  发送    │    │  约定    │    │  成长    │    │  回忆    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │               │               │
     ▼               ▼               ▼               ▼
 邀请码生成      双方确认        任务打卡        关系结束
 发送邀请       电子签名        奖励成就        保留记录
                                可协商调整      可重新开始
```

> **用户自主性体现**:
> - 任何一方可随时发起解除关系
> - 解除后有 7 天冷静期，可撤销
> - 历史记录保留，作为成长回忆

```yaml
services:

      dockerfile: Dockerfile
    environment:
      - DB_PATH=/data/app.db
      - OSS_ENDPOINT=${OSS_ENDPOINT}
      - OSS_BUCKET=${OSS_BUCKET}
      - OSS_ACCESS_KEY=${OSS_ACCESS_KEY}
      - OSS_SECRET_KEY=${OSS_SECRET_KEY}
      - JWT_SECRET=${JWT_SECRET}
    volumes:
      - ./data:/data
    depends_on:
      - redis
    networks:
      - partner-network
    restart: unless-stopped

  redis:
    image: redis:alpine
    volumes:
      - redis-data:/data
    networks:
      - partner-network
    restart: unless-stopped

volumes:
  redis-data:

networks:
  partner-network:
    driver: bridge
```

## 11. 安全与隐私

### 11.1 安全措施

| 类别 | 措施 |
|------|------|
| **认证安全** | JWT Token、密码加盐哈希 (bcrypt)、Token 过期刷新 |
| **传输安全** | HTTPS/SSL 加密、API 请求签名 |
| **数据安全** | 敏感数据加密存储、SQL 注入防护、XSS 防护 |
| **访问控制** | RBAC 权限模型、接口限流、IP 黑白名单 |
| **日志审计** | 操作日志记录、异常日志监控 |

### 11.2 隐私保护

| 措施 | 说明 |
|------|------|
| 数据隔离 | 伙伴关系数据仅双方可见 |
| 约定加密 | 约定内容加密存储 |
| 匿名选项 | 用户可选择匿名显示 |
| 数据导出 | 支持用户导出个人数据 |
| 账号注销 | 支持账号注销和数据删除 |
| 访问密码 | 敏感页面可设置访问密码 |

### 11.3 安全词与边界保护

```
安全词机制：
1. 用户设置黄色/红色安全词
2. 任何任务中，用户输入安全词 → 任务立即暂停
3. 伙伴收到紧急通知
4. 系统记录安全词使用事件

边界保护：
1. 用户设置禁忌事项
2. 伙伴布置任务时，系统自动检测边界
3. 越界任务需要额外确认或禁止发布
4. 用户可对越界任务申诉
```

### 11.4 用户自主权保护

| 权利 | 说明 |
|------|------|
| **关系自主** | 任何一方可随时发起解除关系 |
| **冷静期** | 解除后有 7 天冷静期，可撤销 |
| **任务自主** | 成长者可自主领取任务，不完全依赖布置 |
| **数据自主** | 支持数据导出和账号注销 |
| **申诉权** | 对任务/建议有异议时可申诉 |

## 12. 合规与内容安全

### 12.1 合规原则

| 原则 | 说明 |
|------|------|
| **年龄限制** | 仅允许 18 岁以上用户注册使用 |
| **内容合规** | 禁止违法违规、低俗色情、暴力内容 |
| **用户保护** | 内置安全词、边界设置、紧急暂停 |
| **正向引导** | 强调成长、提升、习惯养成 |
| **透明运营** | 用户协议、隐私政策公开透明 |

### 12.2 敏感词过滤系统

```
敏感词分类：
┌─────────────────────────────────────────────────────┐
│  类别          │  示例          │  处理方式        │
├─────────────────────────────────────────────────────┤
│  政治敏感      │  xxx           │  禁止 + 告警     │
│  色情低俗      │  xxx           │  禁止 + 告警     │
│  暴力恐怖      │  xxx           │  禁止 + 告警     │
│  违法违规      │  xxx           │  禁止 + 告警     │
│  引导规避      │  绕过审核等    │  禁止 + 警告     │
│  竞品推广      │  竞品名称      │  限制 + 提醒     │
└─────────────────────────────────────────────────────┘

处理流程：
1. 用户提交内容（任务/动态/私信）
2. 后端敏感词中间件拦截
3. 匹配敏感词库
4. 根据级别处理（禁止/替换/告警）
5. 记录日志，严重情况通知管理员
```

```
┌─────────────────────────────────────────────────────┐
```

### 12.4 实名认证方案

| 方案 | 说明 | 推荐 |
|------|------|------|
| **基础版** | 手机号验证 + 年龄声明 | ✅ 初期采用 |
| **进阶版** | 身份证实名认证 | 中期采用 |
| **严格版** | 人脸识别 + 身份证 | 后期可选 |
